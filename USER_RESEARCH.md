**调研日期**: 2026-03-06  
**调研方法**: GitHub Issues 分析、社区论坛观察、竞品用户反馈  
**目标**: 理解量化交易用户痛点，指导 v0.4.0 功能规划

---

## 1. GitHub Issues 常见需求分析

### 1.1 vnpy Issues 分析 (12 个开放 Issues)

#### 高频问题类型
| 问题类型 | 占比 | 典型问题 |
|----------|------|----------|
| 安装配置问题 | 35% | "Windows 11 安装失败，依赖冲突" |
| 数据源问题 | 25% | "Tushare 接口调用失败，如何切换数据源？" |
| 策略开发问题 | 20% | "CTA 策略参数优化方法" |
| 实盘交易问题 | 15% | "模拟盘与实盘收益差异大" |
| 其他 | 5% | 文档错误、功能建议 |

#### 关键洞察
- **安装门槛高**: Python 环境配置、依赖冲突是新手最大障碍
- **数据源单一**: 过度依赖 Tushare，缺乏备选方案
- **文档与实践脱节**: 文档示例与实盘场景有差距

## 2. 量化社区痛点分析

### 2.1 学习门槛痛点

#### 痛点描述
```
用户画像：传统交易者/金融从业者，有交易经验但编程基础弱

痛点场景:
1. 想尝试量化交易，但被 Python 编程吓退
2. 看懂策略逻辑，但无法转化为代码
3. 网上策略代码质量参差不齐，不敢直接用
4. 学习周期长 (3-6 个月)，中途放弃率高
```

#### 用户原话 (来自社区论坛)
> "我懂交易，但代码真的搞不定。看了 vnpy 文档，还是不知道从哪里开始。"
> 
> "网上找的策略代码，回测很好，实盘就亏，不知道问题在哪。"
> 
> "想学量化，但 Python+ 机器学习+ 金融知识，太难了。"

#### 机会点
✅ **自然语言策略定义** - OpenFinAgent 核心优势
✅ **策略模板库** - 经过验证的策略，一键使用
✅ **渐进式学习路径** - 从自然语言 → 理解代码 → 修改策略

---

### 2.2 数据获取痛点

#### 痛点描述
```
用户画像：个人量化交易者/小团队，预算有限

痛点场景:
1. 免费数据质量差 (缺失、错误、延迟)
2. 付费数据贵 (Tushare 专业版、Wind、Choice)
3. 多数据源切换麻烦
4. 历史数据下载慢，存储管理复杂
```

#### 数据源对比
| 数据源 | 费用 | 质量 | 覆盖市场 | 用户反馈 |
|--------|------|------|----------|----------|
| Tushare | 免费/付费 | ⭐⭐⭐⭐ | A 股为主 | "积分制度复杂" |
| Yahoo Finance | 免费 | ⭐⭐⭐ | 全球 | "偶尔接口不稳定" |
| Alpha Vantage | 免费/付费 | ⭐⭐⭐ | 全球 | "API 调用次数限制" |
| Wind/Choice | 昂贵 (年费) | ⭐⭐⭐⭐⭐ | 中国全市场 | "机构专用，个人用不起" |
| QuantConnect | 付费订阅 | ⭐⭐⭐⭐⭐ | 全球 | "绑定平台，不灵活" |

#### 机会点
✅ **多数据源集成** - Yahoo Finance + Alpha Vantage 免费方案
✅ **数据缓存机制** - 减少重复下载
✅ **数据质量监控** - 自动检测异常数据

---

### 2.3 回测 - 实盘差距痛点

#### 痛点描述
```
用户画像：有回测经验，准备实盘的交易者

痛点场景:
1. 回测收益很好，实盘就亏损
2. 滑点、手续费模拟不准确
3. 流动性问题未考虑 (大资金冲击成本)
4. 网络延迟、接口限制未考虑
```

#### 回测 - 实盘差距原因分析
| 原因 | 影响程度 | 是否可模拟 |
|------|----------|------------|
| 滑点设置不合理 | ⭐⭐⭐⭐ | ✅ 可改进 |
| 手续费计算错误 | ⭐⭐⭐⭐ | ✅ 可改进 |
| 流动性假设过于乐观 | ⭐⭐⭐ | ⚠️ 部分可模拟 |
| 数据前视偏差 | ⭐⭐⭐⭐ | ✅ 可检测 |
| 网络延迟/断线 | ⭐⭐ | ⚠️ 难模拟 |
| 心理因素 | ⭐⭐⭐ | ❌ 无法模拟 |

#### 机会点
✅ **模拟盘功能** - 用实时数据但不下实单，验证策略
✅ **更精确的回测参数** - 滑点、手续费、冲击成本可配置
✅ **回测诊断工具** - 自动检测前视偏差、过拟合

---

### 2.4 策略分享与协作痛点

#### 痛点描述
```
用户画像：希望学习/分享策略的社区用户

痛点场景:
1. 有好策略，但不知道如何分享变现
2. 想学习他人策略，但找不到可靠来源
3. 策略代码质量参差不齐，难以评估
4. 缺乏策略讨论和反馈机制
```

#### 现有方案对比
| 平台 | 分享方式 | 变现能力 | 质量控制 | 用户反馈 |
|------|----------|----------|----------|----------|
| vnpy 论坛 | 帖子分享 | ❌ | ⚠️ 自发 | "水贴多，干货少" |
| QuantConnect | 平台内分享 | ✅ 订阅分成 | ✅ 平台审核 | "仅限平台用户" |
| GitHub | 开源代码 | ❌ | ⚠️ Star 数参考 | "缺乏金融上下文" |
| 掘金/知乎 | 文章分享 | ⚠️ 打赏 | ❌ | "碎片化，不完整" |

#### 机会点
✅ **策略市场** - 内置策略分享/下载平台
✅ **策略评级系统** - 基于回测表现 + 用户评价
✅ **策略讨论区** - 每个策略独立讨论线程
✅ **收益追踪** - 实盘收益公开透明 (可选)

### 2.5 AI/ML 量化痛点

#### 痛点描述
```
用户画像：想尝试 AI 量化，但技术基础弱的交易者

痛点场景:
1. 听说机器学习很厉害，但不知道如何用
2. 特征工程太复杂，不知道选哪些因子
3. 模型训练调参困难，效果不稳定
4. 过拟合严重，回测好实盘差
```

#### 技术门槛分析
```
AI 量化技能树:
├── Python 编程 ⭐⭐⭐ (必需)
├── 机器学习基础 ⭐⭐⭐⭐ (必需)
├── 特征工程 ⭐⭐⭐⭐⭐ (最难)
├── 模型选择与调参 ⭐⭐⭐⭐ (难)
├── 过拟合检测与防止 ⭐⭐⭐⭐ (难)
└── 金融领域知识 ⭐⭐⭐⭐ (必需)

综合门槛: ⭐⭐⭐⭐⭐ (极高)
```

#### 竞品方案
| 平台 | AI 功能 | 门槛 | 用户反馈 |
|------|--------|------|----------|
| vnpy v4.0 (alpha) | Lasso/LightGBM/MLP | ⭐⭐⭐⭐ | "刚发布，待验证" |
| Qlib | 完整 ML 框架 | ⭐⭐⭐⭐⭐ | "太强大了，但学不会" |
| QuantConnect | 基础 ML 库 | ⭐⭐⭐ | "够用，但不深入" |
| Backtrader | 需自行集成 | ⭐⭐⭐⭐⭐ | "太麻烦" |

#### 机会点
✅ **AI 辅助策略生成** - 用自然语言描述，AI 生成策略
✅ **自动化特征工程** - 内置常用因子，一键生成
✅ **模型推荐系统** - 根据数据特征推荐合适模型
✅ **过拟合检测** - 自动警告可能的过拟合风险

## 4. 用户需求优先级排序

### Kano 模型分析

#### 基本型需求 (Must-be)
> 没有会不满意，有了不一定满意

- ✅ 稳定的回测引擎
- ✅ 准确的数据源
- ✅ 清晰的文档
- ✅ 基础策略模板

#### 期望型需求 (One-dimensional)
> 越好越满意

- ✅ 更多数据源支持
- ✅ 更精确的回测模拟
- ✅ 策略分享功能
- ✅ 模拟盘功能
- ✅ 用户系统

#### 兴奋型需求 (Delighters)
> 有了会惊喜，没有也正常

- ✅ 自然语言策略定义 (OpenFinAgent 核心)
- ✅ AI 辅助策略优化

- ✅ 策略收益追踪

## 6. 关键洞察总结

### 6.1 核心痛点 Top 5

1. **学习门槛高** - 编程 + 金融 +AI 三重门槛
2. **数据获取难** - 免费数据质量差，付费数据贵
3. **回测 - 实盘差距** - 模拟不准确，实盘验证成本高
4. **策略分享难** - 缺乏可靠平台，质量参差不齐
5. **AI 量化难** - 技术门槛极高，过拟合严重

### 6.2 OpenFinAgent 差异化价值

| 痛点 | 竞品方案 | OpenFinAgent 方案 | 优势 |
|------|----------|-------------------|------|
| 学习门槛 | 文档 + 示例 | 自然语言策略 | 大幅降低门槛 |
| 数据获取 | 单一数据源 | 多免费数据源 | 降低成本 |
| 回测 - 实盘 | 无/弱 | 模拟盘功能 | 低成本验证 |
| 策略分享 | 论坛/GitHub | 内置策略市场 | 一站式体验 |
| AI 量化 | 高门槛 | AI 辅助生成 | 降低技术门槛 |

### 6.3 v0.4.0 功能决策依据

**必须做 (P0)**:
- 用户痛点最强烈
- 竞品弱项/空白
- 技术可行性高
- 2 周内可完成

**应该做 (P1)**:
- 用户价值高
- 增强竞争力
- 技术难度中等
- 可分期实现

**可以做 (P2)**:
- 锦上添花
- 技术难度高
- 时间不足可砍掉
