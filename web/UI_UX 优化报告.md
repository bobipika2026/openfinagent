### 1. 优化批量回测进度展示

**文件**: `pages/04_📈_选股引擎.py`

**优化内容**:
- ✅ 添加进度条 `st.progress` 实时显示回测进度
- ✅ 显示当前处理的股票名称和代码
- ✅ 显示已处理数量/总数量 (格式：`正在回测：{symbol} (i/total)`)
- ✅ 支持取消操作（`st.button("⏹️ 取消")`）
- ✅ 进度状态管理（running/completed/cancelled/failed）

**关键改进**:
```python
# 进度条和状态文本
progress_bar = st.progress(0)
status_text = st.empty()

# 实时更新进度
for i, stock in enumerate(stocks, 1):
    progress = i / total
    progress_bar.progress(progress)
    status_text.text(f"正在回测：{stock.get('symbol', 'N/A')} ({i}/{total})")
    
    # 检查取消
    if st.session_state.get('batch_backtest_cancel', False):
        break
```

**用户体验提升**:
- 用户可清晰看到回测进度，不再"盲等"
- 可随时取消长时间回测任务
- 明确的完成/取消/失败状态提示

### 2. 实现实时信号自动刷新

**文件**: `pages/05_🔔_监控中心.py`

**优化内容**:
- ✅ 使用 `st.autorefresh` 组件实现自动刷新
- ✅ 可配置刷新间隔（5 秒/10 秒/30 秒/60 秒）
- ✅ 显示上次刷新时间（精确到秒）
- ✅ 支持手动刷新按钮

**关键改进**:
```python
# 侧边栏配置
refresh_interval = st.sidebar.selectbox(
    "自动刷新间隔",
    [5000, 10000, 30000, 60000],
    format_func=lambda x: f"{x//1000}秒",
    default=10000
)

# 自动刷新
st.autorefresh(refresh_interval)

# 显示刷新时间
st.caption(f"🕒 上次刷新：{st.session_state.signal_last_refresh.strftime('%H:%M:%S')}")
```

**用户体验提升**:
- 信号实时更新，无需手动刷新页面
- 灵活的刷新间隔配置，适应不同场景
- 明确的刷新时间提示，增强可控感

### 3. 增强错误处理

**文件**: 所有 Web UI 页面
- `pages/03_🤖_策略工厂.py`
- `pages/04_📈_选股引擎.py`
- `pages/05_🔔_监控中心.py`

**优化内容**:
- ✅ 统一错误提示样式（使用 `st.error` + Emoji ❌）
- ✅ 提供解决方案链接（`st.expander("💡 可能的解决方案")`）
- ✅ 添加"重试"按钮（`st.button("🔄 重试")`）
- ✅ 错误日志记录指引

**统一错误处理模板**:
```python
try:
    result = some_operation()
except Exception as e:
    st.error(f"❌ 操作失败：{str(e)}")
    with st.expander("💡 可能的解决方案"):
        st.markdown("""
        1. 检查网络连接
        2. 确认数据源可用
        3. 查看日志文件：`~/.openfinagent/logs/`
        """)
    if st.button("🔄 重试"):
        st.rerun()
```

**覆盖场景**:
- 策略创建失败
- 策略库加载失败
- 选股操作失败
- 批量回测失败
- 监控任务创建/启动/停止/删除失败
- 信号加载失败
- 通知历史加载失败

**用户体验提升**:
- 友好的错误提示，减少用户焦虑
- 明确的解决方案指引，降低求助成本
- 一键重试功能，快速恢复操作
