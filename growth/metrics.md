## 四、数据收集方法

### 4.1 自动化工具

#### GitHub API
```bash
# 获取 Star 数量
curl -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/repos/bobipika2026/openfinagent

# 获取 Star 增长历史
curl -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/repos/bobipika2026/openfinagent/stargazers
```

#### 网站统计
- 接入 Google Analytics 或 百度统计
- 追踪页面访问、来源渠道、用户行为
- 设置转化目标 (下载、注册等)

#### UTM 参数
```
示例链接:
https://github.com/bobipika2026/openfinagent?utm_source=juejin&utm_medium=article&utm_campaign=launch

渠道代码:
- utm_source: juejin, zhihu, bilibili, wechat, kol_[name]
- utm_medium: article, video, post, ad
- utm_campaign: launch, strategy_contest, weekly_[date]
```

### 4.3 用户调研

#### NPS 调研 (每季度)
```
问题：您有多大可能向朋友或同事推荐 OpenFinAgent？
评分：0-10 分

- 9-10 分：推荐者 (Promoter)
- 7-8 分：被动者 (Passive)
- 0-6 分：贬损者 (Detractor)

NPS = 推荐者% - 贬损者%
目标：NPS > 50
```

#### 用户访谈 (每月 5-10 人)
- 新用户：为什么选择 OpenFinAgent？
- 活跃用户：最喜欢什么功能？
- 流失用户：为什么不再使用？
- 贡献者：什么激励你贡献代码？

## 六、预警机制

### 6.1 预警指标

| 指标 | 预警线 | 行动线 | 应对措施 |
|-----|--------|--------|---------|
| Stars 日增长 | <10 | <5 | 加大内容投放、检查负面舆情 |
| 社群活跃度 | <100 消息/日 | <50 消息/日 | 策划活动、主动话题引导 |
| 内容阅读量 | <1000/篇 | <500/篇 | 优化选题、改进标题 |
| 用户留存率 | <30% | <20% | 用户调研、产品改进 |
| Issue 响应时间 | >4 小时 | >24 小时 | 增加维护人手、优化流程 |

### 6.2 应对预案

#### Stars 增长放缓
1. 检查是否有负面评价
2. 加大内容投放频率
3. 联系 KOL 推广
4. 策划营销活动
5. 优化 GitHub 页面 (README、示例)

#### 用户留存低
1. 用户调研 (问卷 + 访谈)
2. 分析流失节点
3. 改进 onboarding 流程
4. 加强社群运营
5. 提供更多教程和案例

#### 内容效果差
1. 分析数据 (阅读完成率、分享率)
2. 用户调研 (喜欢什么内容)
3. 竞品分析 (什么内容火)
4. A/B 测试标题和形式
5. 调整内容策略
